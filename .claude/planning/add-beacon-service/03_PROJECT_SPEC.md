# Project Spec: add-beacon-service

**Phase:** 3 (Design) | **Date:** 2026-05-10 | **Risk:** High
**Companion:** 03_ARCHITECTURE.md, 03_ADR-077..082, 00_STATUS.md
**Stack:** Go 1.26.3 / distroless static-debian12:nonroot / GitHub Actions

This spec is the contract between Phase 3 (Design) and Phase 4 (Planning). It enumerates Technical Requirements, Non-Functional Requirements, Interface Contracts, Testing, Migration, Feature Flags, and Rollback. Each TR/NFR is acceptance-test bound.

## 1. Technical Requirements (TRs)

| ID | Requirement | Acceptance test |
|---|---|---|
| TR-1 | Beacon implements `enroll` subcommand: bootstrap-token-then-CSR ceremony per ADR-067. ECDSA P-256 keypair, empty Subject CSR, `POST /api/v1/beacons/enroll`, persists `beacon.crt` (0644) + `beacon.key` (0600) + `dek.bin` (0600) + `platform-pubkey.pem` (0644) atomically via tmpfile + rename. | `internal/enroll/enroll_test.go` against stub server; integration test against live netbrain dev instance. |
| TR-2 | Beacon implements `daemon` subcommand: orchestrates collectors, store, transport, config_poll, probe, metrics, admin. Single root context cancelled on SIGTERM/SIGINT. | `internal/daemon/run_test.go`: start daemon, send SIGTERM, assert clean exit < 30 s. |
| TR-3 | Beacon implements `status` / `collectors {list,enable,disable}` / `logs --tail` subcommands per D-1. Communicates with daemon via Unix socket / Windows named pipe. | `internal/admin/cli/cli_test.go` against running daemon. |
| TR-4 | OpenAPI client generated by `oapi-codegen v2.4.0+` against `services/api-gateway/openapi/beacon-v1.yaml`. Uses OpenAPI 3.1 → 3.0 overlay shim. `go generate ./internal/api/...` is hermetic; CI fails on uncommitted diff. | CI step `go generate && git diff --exit-code`. |
| TR-5 | mTLS HTTP client uses `tls.Config{MinVersion: tls.VersionTLS13, RootCAs: pinnedCAPool, Certificates: ...}`. Stdlib `net/http` only; no third-party HTTP libraries. | `internal/transport/transport_test.go`: dial TLS-1.2-only stub server → connection refused. |
| TR-6 | AES-256-GCM envelope per ADR-068: `[ver(1B=0x01)|dek_v(1B)|iv(12B)|ct|tag(16B)]`. AAD = `bytes([dek_v]) \|\| idempotency_key_bytes` (17 bytes). IVs from `crypto/rand.Read`. **Stdlib only** (`crypto/aes`, `crypto/cipher`). | `internal/crypto/dek_envelope_test.go` + cross-language fixture (ADR-080). |
| TR-7 | UUIDv5 derivation per `idempotency.py`: `uuid.NewSHA1(NS_BEACON_BATCH, []byte(hex.EncodeToString(beaconID[:]) + hex.EncodeToString(sha256.Sum256(plaintext)[:])))`. **Critical:** hex string bytes, NOT raw bytes. | `internal/crypto/idempotency_test.go` + cross-language fixture (ADR-080). |
| TR-8 | ed25519 signature verify (M-11) on rotated DEKs. Header `X-Beacon-DataKey-Signature: <base64>`. Signed payload = canonical JSON of `{beacon_id, data_key_b64, data_key_version, issued_at}` with `sort_keys=true, separators=(",",":"), ensure_ascii=true`. Verify against `platform_pubkey_pem` pinned at enrollment. **On verify-fail:** reject + log + retain old DEK + increment metric. | `internal/crypto/platform_verify_test.go` + cross-language fixture (ADR-080); negative test with tampered signature. |
| TR-9 | Streaming gunzip (M-6): `io.CopyN(buf, gzip.NewReader(blob), maxBytes+1)` pattern; abort on `len > maxBytes`. **Forbidden:** `io.ReadAll(gzip.NewReader(...))`. | `internal/crypto/streaming_gunzip_test.go`: 4 KB encrypted → 100 MB plaintext zeros → abort within 100 ms. |
| TR-10 | Cert auto-rotate at 80% lifetime per ADR-079: tmpfile + `os.Rename` for files; `atomic.Pointer[http.Client]` swap for in-memory client. No mutation of live `Transport.TLSClientConfig`. | `internal/transport/manager_test.go`: simulated rotation with 100 concurrent in-flight requests; race detector clean. |
| TR-11 | Config poll loop: `GET /config` every 60 s ± 10 s with `If-None-Match`. On 200: hot-reload + persist `applied-config.json` + `etag.txt`. On 304: no-op. On 200 with `X-Beacon-DataKey-Signature`: TR-8 then persist `dek.bin`. | `internal/config_poll/poll_test.go` against stub server returning 200/304/DEK-rotation cases. |
| TR-12 | Heartbeat piggybacks: immediately after `/config`, `POST /heartbeat` with current `BeaconHeartbeatRequest` (queue depths, store stats, device probes, `pending_config_hash`). | `internal/config_poll/heartbeat_test.go`. |
| TR-13 | Cert-status check per heartbeat: `GET /cert-status`. If `recommended_action=rotate` OR local 80% crossed → invoke `transport.Manager.Rotate()`. | `internal/transport/cert_lifecycle_test.go`. |
| TR-14 | 4 collectors implemented per D-6/D-9/D-10: syslog (`leodido/go-syslog v4`, 8w/1000q), netflow (`netsampler/goflow2 v2.2.6` + pure-Go nfcapd writer, 4w/500q), snmp (`gosnmp v1.37+`, 16w/200q), configs (`golang.org/x/crypto/ssh`, 4w/100q). | Per-collector unit tests + integration tests against device sims (`snmpsim`, syslog-ng, etc.). |
| TR-15 | bbolt store-and-forward per ADR-078: 4 data buckets (flows/logs/snmp/configs) keyed by UUIDv7; meta bucket with cursors + bytes:type + schema_version. 5 GB / 14 d cap; priority eviction `flows → logs → snmp → never configs`; 2× replay pacing per ADR-071. | `internal/store/eviction_test.go` simulating 5 GB-of-flows + 14-d-old → assert eviction order; integration test for replay pacing. |
| TR-16 | TCP-connect probe per ADR-072: every 5 min per device, ports 22 → 161 → 80 fallback, median-of-3, results in heartbeat. **All dials via `safe_dial`.** | `internal/probe/probe_test.go`. |
| TR-17 | `internal/safe_dial` per ADR-081: resolve once, allow-list-check, dial IP literal. **Forbidden ranges:** `169.254.0.0/16`, `127.0.0.0/8`, `0.0.0.0`, `224.0.0.0/4`, `ff00::/8`, `fe80::/10`. CI lint forbids `net.Dial*` outside this package. | 14 forbidden-range test cases + DNS rebinding integration test + `golangci-lint` gate. |
| TR-18 | Cross-language byte-exactness test fixtures per ADR-080 — 21 cases under `tests/fixtures/cross_lang/`. CI runs them on every build. | TR-19 | Prometheus metrics on `127.0.0.1:9090` per D-8. 17 canonical metrics (see Architecture §4.6). `--no-metrics` opts out. | `internal/metrics/metrics_test.go`: scrape `/metrics`, assert all 17 names + correct types. |

## 3. Interface Contracts

### 3.1 Beacon boot config (YAML on disk — D-2)

`/etc/netbrain-beacon/config.yaml`:

```yaml
# Operator-edited, mode 0644
server:
  url: https://netbrain.example.com:8443
  ca_pem_path: /etc/netbrain-beacon/platform-ca.pem  # bundled at install

state_dir: /var/lib/netbrain-beacon
log_level: info        # debug | info | warn | error
log_destination: file  # file | journald | stdout | windows-event

metrics:
  enabled: true
  bind: 127.0.0.1:9090

admin:
  socket_path: /var/run/netbrain-beacon.sock  # or named pipe on Windows

# Anything beyond this is server-pushed via /config — DO NOT EDIT by hand.
```

Parsed via `gopkg.in/yaml.v3` with strict-decode (struct tags `yaml:"...,omitempty"` for optionals; unknown keys reject).

```go
type BootConfig struct {
    Server       ServerConfig       `yaml:"server"`
    StateDir     string             `yaml:"state_dir"`
    LogLevel     string             `yaml:"log_level"`
    LogDestination string           `yaml:"log_destination"`
    Metrics      MetricsConfig      `yaml:"metrics"`
    Admin        AdminConfig        `yaml:"admin"`
}
type ServerConfig struct {
    URL       string `yaml:"url"`
    CAPemPath string `yaml:"ca_pem_path"`
}
type MetricsConfig struct {
    Enabled bool   `yaml:"enabled"`
    Bind    string `yaml:"bind"`
}
type AdminConfig struct {
    SocketPath string `yaml:"socket_path"`
}
```

### 3.2 Server-pushed runtime config (mirrors `BeaconConfig` in OpenAPI)

Generated by `oapi-codegen` from the spec — Go struct names match schema names. Persisted as `applied-config.json` (0644) for ETag short-circuit + crash recovery.

### 3.3 CLI subcommands (input/output shapes)

```
netbrain-beacon enroll --server-url URL --token TOKEN [--state-dir DIR]
  → exit 0: enrollment success; prints beacon_id
  → exit 1: invalid token / network / verification failure (specific error to stderr)

netbrain-beacon daemon [--config PATH] [--no-metrics]
  → runs forever; exit 0 on graceful SIGTERM; exit non-zero on fatal error

netbrain-beacon status [--json]
  → prints multi-line operator summary OR JSON; exit 0
  → exit 2 if daemon not running (admin socket unreachable)

netbrain-beacon collectors list [--json]
  → table of {name, state, queue_depth, last_success_at, dropped_total}

netbrain-beacon collectors {enable|disable} <name>
  → writes override file consumed by next config-apply tick

netbrain-beacon logs --tail [-n N] [--since DURATION]
  → follows beacon.log (Linux) / Event Log (Windows)

netbrain-beacon version
  → prints "netbrain-beacon vX.Y.Z (git: <sha>, built: <ts>)"
```

### 3.4 On-disk file formats

| File | Format | Notes |
|---|---|---|
| `beacon.crt` | PEM CERTIFICATE | one block; mode 0644 |
| `beacon.key` | PEM EC PRIVATE KEY | one block, ECDSA P-256; mode **0600** |
| `dek.bin` | raw 32 bytes | not PEM-wrapped; mode **0600** |
| `dek.prev.bin` | raw 32 bytes | mode **0600**; deleted after 7-d grace |
| `platform-pubkey.pem` | PEM PUBLIC KEY | SPKI ed25519; mode 0644 |
| `applied-config.json` | JSON | mirrors `BeaconConfigResponse`; mode 0644 |
| `etag.txt` | ASCII string | one line, no trailing newline; mode 0644 |
| `beacon-id.txt` | ASCII UUID | RFC 4122 format; mode 0644 |
| `beacon-state.bbolt` | bbolt | per ADR-078; mode 0600 |
| `known_hosts` | OpenSSH known_hosts | append-only; mode 0600 |

### 3.5 Prometheus metric names

Canonical list per Architecture §4.6 (17 metrics, all `netbrain_beacon_` prefix). Spec-frozen; renames require ADR + dashboard migration.

### 3.6 Environment variables (read at startup)

| Var | Default | Purpose |
|---|---|---|
| `NETBRAIN_BEACON_CONFIG` | `/etc/netbrain-beacon/config.yaml` | Boot config path |
| `NETBRAIN_BEACON_STATE_DIR` | (from config.yaml) | Override state dir |

## 4. Testing Requirements

### 4.1 Unit (per package)

Every `internal/*` package has a `*_test.go` with ≥ 80% line coverage. Total target: ≥ 80% repo-wide. Stdlib `testing` + `stretchr/testify/require`.

Critical packages (≥ 90% required):
- `internal/crypto/*` (every wrap/unwrap, sign/verify, gunzip path)
- `internal/safe_dial/*` (every forbidden range)
- `internal/store/*` (eviction, recovery, cursor advance)

### 4.2 Property-based (`pgregory.net/rapid`)

- `internal/crypto/dek_envelope_property_test.go` — `Wrap(Unwrap(x)) == x` for arbitrary plaintext + dek + idempotency key.
- `internal/store/cursor_property_test.go` — for arbitrary record sequences, cursor advance never skips records.
- `internal/safe_dial/classify_property_test.go` — for arbitrary `net.IP`, classifier matches stdlib `IsLoopback`/`IsLinkLocalUnicast`/etc. unions.

### 4.3 Cross-language byte-exactness (per ADR-080)

21 fixture cases under `tests/fixtures/cross_lang/`. Run on every CI build. Fail message: "Python says X, Go got Y — UUIDv5 derivation regressed."

### 4.4 Integration (against stub netbrain — Phase 5)

- `tests/integration/enroll_test.go` — full enrollment ceremony against stub server; verifies all 4 files persisted with correct modes.
- `tests/integration/daemon_test.go` — daemon runs for 5 min, processes synthetic syslog/flows/snmp/config inputs, verifies all reach the stub server's `/data/*`.
- `tests/integration/cert_rotation_test.go` — daemon receives `recommended_action=rotate` from stub, rotates without dropping in-flight requests.
- `tests/integration/dek_rotation_test.go` — stub returns rotated DEK + signature; verify accepted; tampered signature → rejected.
- `tests/integration/sf_replay_test.go` — daemon offline 5 min, S&F accumulates, daemon back online, replay drains at ≤ 2× normal pacing.

Stub server: a small Go HTTP server in `tests/integration/stub/` mimicking the 9 OpenAPI ops the beacon calls.

### 4.5 End-to-end (against staging netbrain — Phase 7b)

After Phase 7b co-pentest setup:
- Real beacon binary against real staging netbrain.
- Real customer-network simulation (devices in a lab network).
- 24-hour soak test: monitor metrics, assert no drops / leaks / regressions.

### 4.6 Race detector

Every test runs under `go test -race ./...` in CI. Zero races permitted.

### 4.7 Lint gates

`golangci-lint run` with v2 ruleset: `errcheck, gosec, bodyclose, forbidigo, staticcheck, gocritic, revive`. Plus `govulncheck` as a separate hard-fail step.

`forbidigo` rules:
- `math/rand` forbidden in `internal/crypto/**`.
- `cipher.NewGCMWithTagSize`, `cipher.NewGCMWithNonceSize` forbidden everywhere.
- `io.ReadAll(gzip.NewReader(...))` forbidden in `internal/{config_poll,crypto}/**`.
- `net.Dial*`, `(*net.Dialer).Dial*` forbidden outside `internal/safe_dial/**` (one audited exception in `internal/transport/manager.go`).
- `tls.Config{}` literal without `MinVersion` forbidden everywhere.
- `ssh.InsecureIgnoreHostKey()` forbidden.

### 4.8 Benchmark (Phase 5 perf gate)

- `internal/crypto/dek_envelope_bench_test.go` — assert > 500 MB/s AES-GCM throughput on AES-NI hardware.
- `internal/store/store_bench_test.go` — assert > 10k records/s sustained write to bbolt on tmpfs.
- `internal/safe_dial/safe_dial_bench_test.go` — assert < 100 µs allowed dial path (stub resolver).

### 4.9 Mutation testing (deferred to Phase 6)

Optional but recommended for `internal/crypto/*` — `gomutesting` if/when CI budget allows.

## 5. Migration Plan

**N/A — greenfield.** This is a brand-new binary; no prior version exists. The first release ships as v1.0.0.

The platform side (`add-multi-mode-ingestion`) is already at Stage 1 deploy (flag-off); enabling `BEACON_MTLS_ENABLED=true` in staging is gated by this issue's first beacon binary becoming available for the Phase 7b co-pentest. That gate is procedural, not a software migration.

For **future** versions (v1.x → v1.y), see §7 (Rollback) which doubles as forward-compat documentation.

## 6. Feature Flag Strategy

**No feature flags at the binary level.** All gating is server-side via the existing `NETBRAIN_BEACONS_ENABLED` flag in the platform; the beacon respects the resulting `BEACON_PROTOCOL_NOT_ENABLED` 503 by backing off (per Architecture §6.8).

The beacon reads only the four `NETBRAIN_BEACON_*` environment variables listed in §3.6. None are flags; all are operational overrides.

Per-collector enable/disable is **not** a flag — it's runtime config, controlled via:

1. `BeaconConfig` from server (`enabled_collectors: ["syslog", "snmp"]` etc.).
2. Operator CLI (`netbrain-beacon collectors disable syslog`) which writes a transient override consumed by next config-apply.

## 7. Rollback Plan

### 7.1 Downgrade vN → vN-1 (future, not v1)

Procedure once v2 ships:

1. Stop daemon (`systemctl stop netbrain-beacon` / `Stop-Service`).
2. Replace binary with vN-1 (deb/rpm rollback OR manual binary swap).
3. Verify on-disk artifacts are still parseable by vN-1: `beacon.crt`, `beacon.key`, `dek.bin`, `applied-config.json`, `beacon-state.bbolt`.
4. Start daemon. First config poll should re-establish state.

**Compatibility rules going forward:**

- bbolt schema changes require a new `meta:schema_version`; vN-1 with older code rejects newer schema_version → operator must keep both versions of state by rotating `beacon-state.bbolt` to `.upgrade-from-vN.bbolt` before downgrade.
- Wire format changes (envelope ver byte, AAD layout, signature scheme) are coordinated with platform via ADR-068/069 amendments; rolling a beacon vN-1 against a server expecting vN wire format breaks.

For v1 → v2: rollback is in-scope at v2 design time, not now.

### 7.2 Recovery from broken cert / key

Symptom: daemon fails to start with `x509.LoadX509KeyPair` error.

```
1. Check for `beacon.crt.new` / `beacon.key.new` (failed mid-rotation):
   - If both exist + `.bak` files exist: restore from `.bak`.
   - If only one of `.new` exists: discard them; verify the original `.crt`/`.key` parse.
2. If still broken: the cert is genuinely lost.
3. Run `netbrain-beacon enroll --server-url ... --token <fresh nbb_>` to re-enroll.
   Operator must obtain a fresh bootstrap token from the platform UI.
   The OLD beacon record persists on the platform side (admin must DELETE it).
4. The new beacon receives a new beacon_id; old S&F records are NOT replayable
   (different cert → different beacon_id → server rejects on URL mismatch).
```

### 7.3 Recovery from broken / rotated DEK

Symptom: `BEACON_DEK_EXPIRED` 401 storm.

```
1. Verify `dek.bin` and `dek.prev.bin` both present + 32 bytes each.
2. If `dek.bin` missing or wrong size: trigger out-of-cycle GET /config — server sends rotated DEK + signature.
3. Verify ed25519 (M-11). If verify fails: alert; this is a tampering signal.
4. If verify succeeds: persist new dek.bin; resume.
5. If server side has rotated past 7-day grace: re-enroll (§7.2).
```

### 7.4 Recovery from bbolt corruption

Symptom: daemon log "page X: invalid" or panic in bbolt code.

```
1. Stop daemon.
2. mv beacon-state.bbolt beacon-state.bbolt.corrupt.<RFC3339>
3. Start daemon. Fresh bbolt is created automatically.
4. Data loss = whatever was in the corrupt file.
5. Operator can forensic the .corrupt file with `bbolt check beacon-state.bbolt.corrupt.<...>`.
6. Increment `sf_corruption_recovery_total` is automatic.
```

### 7.5 Recovery from cross-language byte-exactness regression

Symptom: `BEACON_IDEMPOTENCY_KEY_MISMATCH` storm; **fatal** halt per Architecture §6.10.

```
1. Pull current beacon binary.
2. Run `go test ./internal/crypto/... -run TestCrossLang` against tests/fixtures/cross_lang/.
3. If the test fails: regression in this beacon build. Roll back to previous version (§7.1).
4. If the test passes: regression is server-side. Coordinate with platform team.
5. File a P0 incident. Pause deploy.
```

### 7.6 Wholesale recovery (operator panic button)

Last-resort recovery: nuke everything and re-enroll fresh.

```
1. systemctl stop netbrain-beacon.
2. mv /var/lib/netbrain-beacon /var/lib/netbrain-beacon.broken.<RFC3339>
3. Operator gets fresh bootstrap token from platform UI.
4. netbrain-beacon enroll --server-url ... --token nbb_<new>
5. systemctl start netbrain-beacon.
6. Lost data = whatever was in S&F at the moment of step 1.
7. Old beacon record on platform side: admin must DELETE it via /api/v1/beacons/{old_id}.
```

## 9. Pending cross-issue dependency

**Phase 7b co-pentest with `add-multi-mode-ingestion` is mandatory** within 7 days of staging deploy of beacon v1. Carries forward from parent issue per `00_STATUS.md` and `01_DISCOVERY.md`. Test coverage requirements:

- H-1: nginx REPLACE semantics (verified empirically by real Go client).
- H-2: cross-beacon access (beacon-A cert + URL `{B_id}` → must 403).
- H-4: token replay across IPs after failed enrollment.
- M-6: gzip bomb at every `/data/*` endpoint.
- M-9: SSRF probe to forbidden IPs (each of 6 ranges).

The pentest report is incorporated into Phase 7b artifact `07b_PENTEST_REPORT.md` for **both** issues; deploy stage 3+ on the platform is blocked until the report is APPROVED.
