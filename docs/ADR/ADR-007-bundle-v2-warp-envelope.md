# ADR-007: Bundle v2 + WARP credential envelope (Go reader)

**Status:** Accepted
**Date:** 2026-05-14
**Context issue:** add-cloudflare-mesh-onboarding (cross-repo)
**Pairs with:** [netbrain/ADR/ADR-087-bundle-v2-encrypted-warp-credential-envelope.md](https://github.com/velonet/netbrain/blob/main/.claude/planning/add-cloudflare-mesh-onboarding/03_ADR-087-bundle-v2-encrypted-warp-credential-envelope.md) — canonical decision lives netbrain-side; this ADR records the Go-reader half of the contract.

## Context

The NetBrain platform's `add-cloudflare-mesh-onboarding` workflow replaces the v1 enrollment bundle with a v2 bundle that carries optional Cloudflare WARP Service-Token credentials in an AEAD envelope. The platform-side decision (Argon2id KEK + AES-256-GCM AEAD + AAD-binding) is locked in netbrain ADR-087.

This ADR is the Go-reader pair: how the beacon parses, version-discriminates, and decrypts v2 bundles, and how byte-exactness with the Python writer is enforced.

## Decision

### Parser

`internal/enroll/bundle_v2.go` exposes `BundleV2` + `ParseBundleV2`. `internal/enroll/bundle.go` keeps the legacy export names — `Bundle` is a type alias for `BundleV2`, `ParseBundle` delegates to `ParseBundleV2` — so pre-cutover callers in the beacon (and downstream Go callers if any) keep compiling with zero churn.

The discriminator behaves as follows:

- `version == 2` → continue parsing as v2.
- `version != 2` or missing → return `ErrBundleVersionUnsupported`. Pre-cutover (v1) bundles surface this clean error, NOT a cryptic "signature invalid" or "missing field".

The Ed25519 signature covers the full v2 payload (every field except `signature` and `platform_pubkey_pem`). A regression test (`TestParseBundleV2_TamperedWARPFields_FailsSignature`) injects mesh fields into a previously-mesh-off bundle to confirm the signature catches the swap — this is the explicit defense against an attacker silently dropping or injecting WARP enrollment hints.

### Crypto

`golang.org/x/crypto/argon2.IDKey` derives the 32-byte KEK at the locked parameters (`t=2 / m=64 MiB / p=1 / len=32`). `crypto/aes` + `crypto/cipher.NewGCM` perform the AEAD decrypt. Both are stdlib + the x/crypto extension — no third-party crypto.

The AAD binding is canonical-JSON of `{beacon_token_prefix: <token[:12]>, expires_at: <iso8601>}` — matching the Python side via `bcrypto.CanonicalizePayload`. The 12-character prefix mirrors Python's `BeaconRepository._token_prefix`.

### Cross-language byte-exactness

`internal/enroll/bundle_v2_test.go::TestCrossLangBundleV2` loads the five bundle-v2 envelope cases from `tests/fixtures/cross_lang/cross_lang_fixtures.json` (regenerated by `scripts/beacon/generate_cross_lang_fixtures.py` in the netbrain repo) and asserts:

1. The locked Argon2id parameter set from the fixture meta matches the Go constants exactly.
2. Given `(token, salt, iv, plaintext, aad)`, the Go writer produces byte-identical envelope output to the Python writer.
3. Python-emitted envelopes round-trip cleanly through the Go decryptor.
4. The `tampered_salt_diff_output` case produces different bytes from `happy` — proves the salt actually flows through the KDF.

The test follows the existing ADR-004 cross-language fixtures pattern. Byte drift is a fail-fast regression gate that catches "someone bumped Argon2id memory cost for faster CI", "someone reordered canonical-JSON keys", or any other unauthorized wire-shape change in either repo.

## Alternatives Considered

| Option | Pros | Cons |
|---|---|---|
| **A: Keep `bundle.go` v1 alongside `bundle_v2.go` v2** | Zero churn on existing tests | Two parsers to maintain; v1 codepath becomes dead code (platform always emits v2) |
| **B: Replace v1 entirely** | One parser, simpler module | Existing callers (enroll command, integration tests) need plumbing churn |
| **C: Type alias + delegating ParseBundle (this ADR)** | One parser; zero call-site churn; clean v1 rejection via discriminator | Slightly indirect — readers have to follow `Bundle = BundleV2` |

Picked **C** — minimizes the diff while still hard-cutting over to v2 semantics.

## Consequences

### Positive

- Cross-language fixtures lock the parser-writer contract; byte drift fails CI.
- `ErrBundleVersionUnsupported` gives operators a clean error message when they hand the beacon a pre-cutover bundle.
- The signature covers all v2 fields including `warp_*`, so a network MITM cannot silently strip or inject mesh enrollment fields.
- `Bundle = BundleV2` type alias preserves existing call sites — Phase 4 was zero touch on `enroll.Input`, `enroll.Enroll`, persist, etc.

### Negative

- Beacon enroll wall-time grows by ~1-3 s on mesh-on bundles (the Argon2id KDF run). Operators see a progress message so they don't think the binary hung.
- New `x/crypto/argon2` dependency. The package is small (stdlib-extension), widely used, and pinned via `go.mod`.

### Risks

- **Argon2id default tweaks between library versions.** Go's `x/crypto/argon2.IDKey` is RFC 9106 with the version byte locked at `0x13`; `cryptography.Argon2id` is the same. The byte-exactness fixtures catch any divergence on first regression. **Mitigation:** the fixture test runs in CI on every commit.
- **AEAD failure ambiguity.** `DecryptWARPEnvelopeRaw` returns `ErrWARPEnvelopeAuthFailed` for ALL failure modes (wrong token / tampered envelope / wrong AAD) — by design we don't distinguish (standard AEAD discipline). Operators only see "envelope decrypt failed". **Acceptable:** the runbook explains the most likely cause is mismatched bundle ↔ token.
